## Cómo Usar Esta Documentación

### Inicio Rápido
1. **Nuevo en el proyecto**: Lee `README.md` (este archivo)
2. **Para empezar a desarrollar**: Lee `DESARROLLO.md` 
3. **Guía rápida de estructura**: Lee `ESTRUCTURA_README.md`
4. **Referencia rápida de comandos**: Consulta `QUICK_REFERENCE.md`

### Búsqueda y Navegación
- **Buscas un componente específico**: Usa `ESTRUCTURA_ARCHIVOS.md`
- **Necesitas ver la estructura visual**: Consulta `ESTRUCTURA_VISUAL.md` y `ESTRUCTURA_COMPLETA.md`
- **No sabes por dónde empezar**: Lee `GUIA_DOCUMENTACION.md` (matriz de decisión)

### Desarrollo de Contenido
- **Agregar nuevas preguntas**: Sigue `CONTENIDO_PREGUNTAS_GUIA.md`
- **Entender formato de datos**: Lee `ESTRUCTURA_DATOS_PREGUNTAS.md`
- **Ver ejemplos de código**: Abre `EJEMPLOS_PREGUNTAS_AVANZADAS.js`

### Entendimiento Profundo
- **Arquitectura del proyecto**: Lee `ESTRUCTURA_NUEVA.md`
- **Servicios y Firebase**: Consulta `ARQUITECTURA_SERVICIOS.md`
- **Diseño y estilos**: Lee `STYLES_GUIDE.md` y `DESIGN_IMPROVEMENTS.md`

## Notas Importantes

- El proyecto usa arquitectura Feature-Based con Atomic Design
- Todos los imports deben usar el alias `@/` (ej: `@/features/exam/pages`)
- Firebase está configurado para desarrollo y producción
- Los datos de preguntas se encuentran en `src/data/`
